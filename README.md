# Triangulum Daily 3 Albums

[English](https://github.com/qianqian5774/triangulum-daily3albums/blob/main/README.en.md) | 中文

每天三张专辑。
不追热点，不刷榜单，不急着听完世界。
只是给一天留三个入口，看看声音会把人带到哪里。🎧

## 🌗 这是什么

**Triangulum Daily 3 Albums** 是一个每天推荐三张专辑的音乐项目。

它最初只是一个很私人的想法：我想给自己一个每天听专辑的入口，让听歌这件事从算法信息流里稍微退出来一点。不是无限滑动，不是即时爽感，也不是“猜你喜欢”的自动投喂，而是每天固定出现三张专辑，像三扇小门。

后来它慢慢变成了一个可以发布到网上的静态网站。每天自动生成内容，按时间解锁，保留归档。你可以每天打开看一眼，也可以隔一段时间回来翻翻过去出现过的专辑。

## ✨ 它会做什么

每天，网站会给出三张专辑推荐。

它们不会一次全部出现，而是按北京时间分成三个时段慢慢解锁：

| 时间  | 内容       |
| ----- | ---------- |
| 06:00 | 第一张专辑 |
| 12:00 | 第二张专辑 |
| 18:00 | 第三张专辑 |

在一天开始之前，页面会保持等待状态。
到了对应时间，新的专辑会自然出现。

这不是一个追求“最热门”“最权威”“最懂你”的推荐系统。它更像一个小型音乐日历：每天翻开一页，可能遇到熟悉的名字，也可能撞见一张完全没听过的唱片。

## 🧭 为什么做它

现在听音乐很方便，但也很容易被平台推着走。

你可能一直听同一批歌手，或者被同一种推荐逻辑困住。Triangulum Daily 3 Albums 想做的事情很简单：每天给你一点轻微的偏离。

不一定冷门，也不一定古怪。
但希望它至少有一点未知感。

它关心的是：

- 一天只给三张，少一点信息噪音。
- 尽量避开过于熟悉、过于主流的结果。
- 保留每天的归档，让推荐变成一条可以回看的时间线。
- 让音乐发现这件事更像散步，而不是刷屏。

## 🕰 每日节奏

Triangulum Daily 3 Albums 使用北京时间作为一天的节奏。

```text
00:00 - 05:59   今日内容尚未开放
06:00 - 11:59   第一张专辑
12:00 - 17:59   第二张专辑
18:00 - 23:59   第三张专辑
```

这个节奏不是为了制造神秘感，而是为了让网站慢下来。

你不需要一次看完所有内容。
一天中有三个时刻，可以停一下，听一张专辑，或者只是把它存下来。

## 🗂 网站内容

网站主要包含：

- **今日推荐**：当天三张专辑，按时间解锁。
- **专辑信息**：艺术家、专辑名、封面和相关资料。
- **历史归档**：过去每天出现过的推荐。
- **详情页面**：查看某一张专辑的单独信息。

后续正式域名上线后，它会作为一个独立的每日音乐站点运行。

## ⚙️ 它背后如何运行

Triangulum Daily 3 Albums 背后有一个自动生成系统。

每天清晨，系统会根据配置、缓存和外部音乐数据源，生成当天的三张专辑，并发布成静态网页。网站本身不依赖实时后端服务，访问者看到的是已经生成好的页面和数据。

简单说：

```text
音乐数据 → 筛选生成 → 静态网站 → 每日访问
```

这种方式的好处是轻、稳定，也更适合长期运行。

## 🌐 发布计划

这个项目最终会发布到我自己的域名上，面向国内外访问者。

目前仓库主要记录网站生成、发布和维护所需的实现。它不是一个面向通用复刻的开源模板，也不是为了让每个人都在本地部署一套同样的系统。

更重要的是网站本身：
每天三张专辑，稳定出现，慢慢积累。
